# Systems Architecture — SYSEN 5400

## Project Overview

This repository contains systems architecture artifacts for SYSEN 5400 coursework, including SysML models, dependency matrices, and supporting analysis. The work follows INCOSE systems engineering practices for stakeholder-driven requirements and architectural decomposition.

## System Scope and Objectives

- Define system boundaries and interfaces through context analysis
- Decompose system functions and allocate to logical architecture
- Capture dependencies via Design Structure Matrix (DSM) for integration sequencing
- Support trade studies and architectural decision documentation

## Methodology

| Phase | Artifact | Description |
|-------|----------|-------------|
| Stakeholder Analysis | Requirements, Use Cases | Identification of stakeholders and derived needs |
| Functional Decomposition | Activity Diagrams, Use Cases | Breakdown of system functions and flows |
| Context Diagram | IBD, Block Definition | System boundary and external interfaces |
| Logical Architecture | Block Definition, IBD | Logical components and allocations |
| DSM | DMM Matrix | Dependency mapping for integration and change impact |
| Trade Studies | Analysis Reports | Evaluation of design alternatives |
